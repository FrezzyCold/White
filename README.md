# Whitecore Client

Сайт + (опционально) Telegram‑бот для раздачи файла чита Whitecore. Есть регистрация/вход с капчей, админ‑панель для загрузки ZIP и превью, а скачивание доступно только авторизованным.

## Быстрый старт

```bash
npm install
npm start
```

- Откройте `http://localhost:3000`
- Админ по умолчанию: **RURI / Wat3hahak**
- Админка: `http://localhost:3000/admin`

## Фичи

- Регистрация и вход с картинкой‑капчей
- Закрытое скачивание: без аккаунта перекинет на страницу входа
- Админ‑панель: загрузка картинки чита и ZIP‑файла
- Telegram‑бот (опция): `/login <логин> <пароль>`, `/download`, `/logout`

## Настройки окружения

Создайте `.env` при необходимости:

```
PORT=3000
SESSION_SECRET=замените-на-секрет
TELEGRAM_BOT_TOKEN=токен_бота # оставить пустым, если бот не нужен
DB_PATH=./data.sqlite
```

## Структура

- `server.js` — сервер Express + SQLite + сессии
- `views/` — EJS шаблоны
- `public/` — стили и скрипты фронта
- `uploads/` — загруженные админом картинки (игнор в git)
- `downloads/` — архив чита; по умолчанию лежит `whitecore-default.zip`

## Telegram‑бот

- Укажите `TELEGRAM_BOT_TOKEN` в `.env` и перезапустите сервер
- Команды: `/start`, `/login <логин> <пароль>`, `/download`, `/logout`
- Бот использует ту же БД и проверяет пароль через bcrypt

## Примечания

- Капча обновляется кнопкой «Обновить» в формах входа/регистрации
- Файлы из `uploads/` и новые ZIP не хранятся в репозитории (см. `.gitignore`)
